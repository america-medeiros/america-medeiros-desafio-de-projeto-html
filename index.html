Antes do HTML

 (voltar para topo)

Antes de falarmos sobre HTML, é importante conhecer os pontos a seguir:

  • Client: dispositivo que faz pedidos à um servidor.
  • Server: equipamento que presta serviços aos dispositivos conectados a ele através de uma rede.

Ou seja, uma conexão Client x Server nada mais é do que a conexão entre um dispositivo com acesso à alguma rede e um servidor.

Mas, América, o que significa isso? Caaalma, pequeno gafanhoto! (Pegue a referência ao melhor professor do mundo).
Pense no Client sendo o cliente de uma lanchonete, a rede de conexão sendo um atendente e o servidor sendo uma cozinha.
Esse cliente fará um pedido ao atendente, nossa rede, que então irá buscar o pedido na cozinha, nosso servidor, o atendente então retornará com o pedido para o cliente.
Essa é, basicamente, a conexão Client x Server.
Alguns servidores ficam entre o client e a conexão de rede e outro ficam entre a conexão de rede e o servidor.

Existem alguns tipos de servidores, entre eles:

  • Proxy:
    É um bloqueador de endereços web. Fica entre o client e a rede.
    Funciona como uma parede entre o Client e alguns tipos de sites pré-selecionados, impedindo o acesso do client aos mesmos.

  • Firewall:
    Também é uma parede de bloqueio entre a rede e o client, mas dessa vez para impedir que arquivos maliciosos entrem no client.

  • Application Server / Web Server:
    São responsáveis pelo acesso do client aos seus app e sites, respectivamente.
    É esse quem faz a conexão de client com o servidor necessário. Nosso querido atendente.
    Quando contratamos um serviço de hospedagem, estamos contratando um espaço em um web server.
    Uma aplicação web (site) não é o mesmo que um aplicativo. Ambos têm semelhanças, mas são diferentes em seu core.

  • Database Server:
    São os servidores de bancos de dados, onde encontramos os arquivos de texto, fotos, vídeos, etc, cada qual em seu respectivo banco de dados.

  • Intranet:
    Chamamos de Intranet aquele servidor sem acesso à internet.
    Seu acesso só é possível de forma local por uma rede de dispositivos exclusiva.
    Geralmente usado por sistemas empresariais para compartilhar recursos internos.

  • DNS:
    É aqui que ficam armazenados os nome e ips dos clients e servidores. DNS significa (Domain Name Service).
Além dos servidores também é bom sabermos o que significa certificado SSL e a sigla FTP, que serão muito uteis na contratação de uma hospedagem para o seu site.
  • Certificado SSL:
    O tal 's' no final de 'https'. Esse certificado cuida para que toda a conexão client x server seja segura
    criptografando esse acesso de ponta a ponta e impedindo a extração de dados em tentativas de intercepção da conexão.

  • FTP:
    Conexão direta entre client e servidor para a transferência de arquivos. Esse acesso é dado pela empresa de hospedagem quando contratamos seu serviço.

Estrutura do HTML

 (voltar para topo)

A estrutura do HTML é sua espinha dorsal. Sem ela, nada funciona. Um código HTML bem estruturado, além de permitir um melhor funcionamento, também transforma a vida
de um desenvolvedor muito mais fácil caso o mesmo precise corrigir algum bug em seu código.

A estrutura do HTML é bem simples e fácil de entender.
nossa estrutura começa abrindo o HTML e fechando o mesmo com tags (Falaremos sobre tags mais a frente, mas você pode clicar aqui se preferir).
Dentro desse HTML aberto, colocamos a head e o body (cabeçalho e corpo) do nosso site.
Tudo o que está na área da head é o que será carregado antes de exibir o site para o usuário e tudo o que está no body é o que irá aparecer para o usuário.

Tags, o que são e para o que servem

 (voltar para topo)

As Tags são, em sua base, os comandos do HTML. são elas quem dirão o que cada parte do site fará ou terá, o tamanho, a fonte, tudo...
A Tags são o sistema nervoso de um site, sem elas nada existe.
Toda tag, salvo algumas exceções, fica entre '< >', sem espaços entre eles e deve ser aberta e fechada
Para as Tags que precisam ser fechadas usamos a '/' após o sinal de '<' e antes da tag.
Exemplo de uma tag:

são exemplos de tags:

  • < i > - Deixa seu texto em itálico;
  • < strong > - Deixa seu texto em negrito;
  • < small > - Deixa seu texto menor;
  • < img > - Tag que permite inserirmos uma imagem em nosso site com o auxílio de alguns atributos;
  • < title > - Tag para usarmos no Head e colocarmos uma título em nossa página
  • < input > - Tag que permite a inserção de dados pelo usuário, também utilizado com atributos;

Enfim, são muitas as tags, algumas menos populares, como a < sub > quer serve para alinhar o texto selecionado com a linha debaixo assim -> olá!
ou a tag < sup >, que faz o oposto e serve para as famosas exponênciações -> 24
ou até mesmo a < blockquote > que nos permite fazer citações em nosso texto, deixando-a em destaque do resto do texto.
Cada tag tem suas especificações e utilidades e todas são muito necessárias para o jovem programador.
No peóximo tópico vamos falar sobre os atributos e suas funções.

Atributos, mais que apenas aliados

 (voltar para topo)

Nesse curto tópico falaremos sobre o que são atributos.
Atributos nada mais são do que as propriedades que uma tag pode ter. Existem aqueles que são globais, que podem ser usados em praticamente
todas as tas e também alguns específicos para algumas tags.

Vamos para alguns exemplos:

  • id - é um identificador de elemento, sendo utilizado para referenciar partes que desejamos no texto com hyperlinks;
  • style - serve para formatar os elementos do HTML com CSS;
  • class - serve para padronizar grupos de elementos com o CSS;
  • src - atributo da tag img onde colocamos o caminho (local ou na web) de onde ele irá pegar a imagem que desejamos.
  • width - atributo que define a largura dos objetos em tela, usada com a tag img também;

Existem ainda outros inúmeros atributos, como type para a tag input, que nos permite definir o tipo de dados que o usuário poderá inserir, entre outros.

Listas, para os amantes da organização

 (voltar para topo)

Sabe as listas acima, de tags, atributos e servidores? Então, elas foram feitas com essa camaradinha aqui.
As listas são muito úteis para organizar sua página na web, podendo ser listas ordenadas < ol >, que são listas com um padrão numérico iniciado em 1,
para quem necessita de uma lista de prioridades, ou então listas não ordenadas < ul >, que em vez de números terão marcadores.
As listas são abertas com a tag de acordo com o tipo desejado (< ol > ou < ul >) e entre elas precisamos de uma tag chamada < li > (list item), na qual colocaremos
nossos items.
Abaixo temos um exemplo de cada:

  1. item 1 (< li >item 1< /li >)
  2. item 2
  3. item 3
  • item 1 (< li >item 1< /li >)
  • item 2
  • item 3

Deus no céu, ancoragem na Terra

 (voltar para topo)

Sabe aqueles campos no meio do texto com um 'clique aqui' que você clica e vai para outra página da web?
Aqueles hyperlinks que facilitam nossa vida quando estamos pesquisando algo específico em um texto?
Eles são feitos com a ancoragem!
Para linkar uma página no texto usamos a tag < a > com o atributo href, isso fará a frase ou palavra dentro dessa tag ficar azul e ser clicável,
levando o usuário para outra página ou abrindo outra página naquela mesma aba de navegação. Se usarmos o atributo target junto, podemos escolher se o link
será aberto em outra página ou se será aberto naquela mesma aba que já estamos navegando.
Uma palavra ou frase com hyperlink é feita assim no HTML -> < a href="link da página" >Frase ou palavra a ser clicável< /a >.
